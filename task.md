### Задание по React

Нужно сделать одностраничное приложение, которое помогает найти ревьюера.

## ✅ Что нужно реализовать

### React часть:

1. **Настройки (Settings)**

    - Кнопка для показа/скрытия настроек
    - 3 поля ввода:
        - `login` - логин текущего пользователя
        - `repo` - репозиторий (формат: `owner/repo`)
        - `blacklist` - список логинов через запятую/массив
    - Сохранение в `localStorage`

2. **Поиск ревьюера**

    - Кнопка "Найти ревьюера"
    - Запрос к GitHub API: Дока по API https://docs.github.com/en/rest.

    - Фильтрация:
        - Исключить текущего пользователя (`login`)
        - Исключить пользователей из `blacklist`
    - Случайный выбор из оставшихся

3. **Визуализация процесса**

    - Показать текущего пользователя
    - Показать анимацию перебора кандидатов
    - Показать выбранного ревьюера

4. **Требования**
    - Только функциональные компоненты
    - Использовать хуки (useState, useEffect, useCallback и т.д.)

### Redux часть:

Добавить redux в приложение для управления состоянием приложения. Сохранять в store сущности ответов от API. Можно использовать RTK

## ⚠️ Важные моменты

### GitHub API:

-   **Rate limit**: 60 запросов/час без авторизации, 5000 с токеном
-   **Пагинация**: по умолчанию 30 результатов, можно указать `?per_page=100`

## 🚨 Потенциальные проблемы

1. **Пустой список контрибьютеров** - обработать случай, когда после фильтрации никого не осталось
2. **Ошибки API** - репозиторий не найден, нет доступа, rate limit
3. **Валидация** - проверить формат `repo` (должен быть `owner/repo`)
